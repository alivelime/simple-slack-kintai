# single-review — 独立シングルレビュー (別コンテキスト × 1体)

このスキルは、比較デモの **2本目**。self-review が残した確証バイアスを、**文脈を共有していない1体のサブエージェント** が客観視することで補完できる、というパターンを示す。

## 前提

- `report/00_internal.md` が存在する (self-review 完了済み)。存在しなければ、ユーザーに「先に `/self-review <機能名>` を実行してください」と伝えて停止する。

---

## 実行手順

### Step 1 — イントロ表示 (YouTube視聴者向け)

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎬 Pattern 2 / 4 : 独立シングルレビュー
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【前提条件】
・Pattern 1 (self-review) が完了している
・report/00_internal.md が存在する

【実行内容】
1. サブエージェントを 1 体起動する
2. サブエージェントは「実装者の思考履歴を持たない」状態で
   コードレビューを実行する
3. そのうえで report/00_internal.md も読み、
   セルフレビューで見落とされていた観点を指摘する
4. 結果を report/01_single.md に保存する

【期待される効果】
✓ 実装者の思い込みがサブエージェントには伝染しない
✓ セルフレビューが拾えていなかった観点が浮かぶ
✓ 「独立コンテキスト」の価値がセルフレビューとの差分で可視化される

【このパターンの弱点】
✗ 1体ぶんの視点しかない (もう1体いたら違う指摘が来るかも)
✗ そのレビュー自体が妥当かを検証する仕組みがない

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 2 — サブエージェントを起動する

`Agent` ツールで **general-purpose** の独立エージェントを1体起動する。

重要な点:
- **会話履歴・セルフレビューの判断を事前に渡さない**。エージェントが自分で report/00_internal.md を読みにいく形にする
- これは「独立コンテキスト」の演出として意図的

プロンプトの骨子 (そのまま渡す):

```
あなたは独立したコードレビュアーです。このプロジェクトの実装者とは
別人格として、フレッシュな視点でコードレビューしてください。

## 手順
1. プロジェクトルートで `git diff HEAD~1..HEAD` または `git status` を確認し、
   直近で変更されたファイル群を把握する
2. CLAUDE.md を読んでプロジェクトの方針を理解する
3. 変更ファイルを読んでコードレビューする
4. 最後に report/00_internal.md (実装者によるセルフレビュー) を読む
5. セルフレビューが **拾えていない観点 / 楽観的すぎる判断 / 見落としている欠陥**
   を指摘する

## レビューの観点
- 機能要件の充足度
- バグ / 型安全性 / 例外処理 / 境界値
- セキュリティ (認証・認可・RLS・入力検証)
- パフォーマンス
- テスタビリティ
- プロジェクト規約への適合

## 出力
report/01_single.md に以下のテンプレートで保存してください。

---

## メタ情報
- レビュー種別: 独立シングルレビュー (別コンテキスト × 1体)
- レビュアー: サブエージェント (実装コンテキスト非共有)
- 入力: 変更コード + report/00_internal.md

## 対象機能
(report/00_internal.md から引用)

## 独立レビューの所見
### 🔴 重大な指摘
### 🟡 改善推奨
### 🟢 良い点

## セルフレビューとの差分
### セルフレビューが拾えていなかった点
- ...

### セルフレビューが過剰に楽観視していた点
- ...

### セルフレビューと一致した点
- ...

## 結論
(merge可否 / 追加対応すべき項目)
---

保存できたら、報告として「report/01_single.md を保存した。主な指摘は
(3行サマリ)」と返してください。
```

メインエージェントはこの Agent 呼び出しの完了を待ち、結果を受け取る。

### Step 3 — 終了時のサマリ

```
✅ Pattern 2 完了: 独立シングルレビュー

レポート: report/01_single.md
独立レビューで追加発見された主要観点:
・(サブエージェント報告から抜粋 3行)

セルフレビューとの比較:
・セルフが見落としていた点の数: N
・セルフが楽観視していた点の数: M

次のステップ: /cross-review を実行して、2体のエージェントに
  相互レビュー → 合意形成をさせる
```

---

## 重要な方針

- **サブエージェントは1体だけ**。ここで2体以上呼ぶと cross-review との差が消える
- **事前知識を渡さない**。エージェントが自分で状況を読み取る演出が「独立コンテキスト」の核
- サブエージェントの結果がそのまま report/01_single.md になる (メインでの書き換えは最小限)
